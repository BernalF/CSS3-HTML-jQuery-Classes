LESSON 2 - CSS AS A MODIFIER

By Caleb Briancesco 07/23/13 at 10:01 am

HTML, HTML5, CSS

CSS RESET
The goal of a reset stylesheet is to reduce browser inconsistencies in things like default line heights, margins and font sizes of headings, and so on.
All browsers have differences between each other in matter of how the display or render each element or even interpret attributes.
With the reset.css we can take all those values and set them to 0 by making all mayor browsers see the same thing or at least the most similar perception of the elements in the HTML.

 

About simple structures

Header tag (<header>):
In this tag you can include navigation if needed; the navigation should be hosted into a nav tag (<nav>).

Anchor (<a>):
An anchor should only exist into a text container such as a p, li, table, titles (h1, h2, h3, h4, h5, h6), and so on.

List Item (<li>):
The li should NEVER be out of an ul (unordered list) or an ol. At the same time the UL or the OL should NEVER EVER hold any element but an LI.

 

About behavior of elements

Inline elements:
Generally, inline elements may contain only data and other inline elements. By default, inline elements do not begin with new line.
Inline elements naturally will only grow according to the amount of content inside them.
These are inline elements:

b, big, i, small, tt, abbr, acronym, cite, code, dfn, em, kbd, strong, samp, var, a, bdo, br, img, map, object, q, script, span, sub, sup, button, input, label, select, textarea

 

Block elements:
Their most significant characteristic is that they typically are formatted with a line break before and after the element (thereby creating a stand-alone block of content). By default, block-level elements begin on new lines.
Block elements will grow horizontally according to the free space they have.

Generally, block-level elements may contain inline elements and other block-level elements. Inherent in this structural distinction is the idea that block elements create "larger" structures than inline elements.

CSS TAKES CONTROL!
We can also change the way an element behaves using CSS, we can create an inline element and make it work as a block element and the other way around.

The inline elements wonâ€™t be able to change their size unless we make them work as a block or as an inline-block; the size restriction will always work on block elements.
If we make a block element float, it will behave as an inline block element growing according to the amount of content it holds and also being able to change its size by request (adding specific size on the CSS).
